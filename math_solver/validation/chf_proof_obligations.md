# CHF Proof Obligations

## Assumptions

1. The Consequence Horizon Formalism starts as a pre-code mathematical sandbox.
2. The first validation target is deterministic and offline.
3. The three initial specs validate transition grammar before any 3D, cosmological, or external-model work.

## PO-001 — Radial Coverage

If a transition region is compact and star-shaped with respect to its operational center, then boundary patches can form inward radial cells that cover the region.

## PO-002 — Category Change

A proposed transition cannot become an actual next state unless a threshold crossing occurs.

## PO-003 — Conservative Uncertainty

If multiple plausible centers remain, robust ALLOW requires the transition to pass under every represented plausible center.

## PO-004 — Local-Global Coupling

A locally allowed transition can be globally denied if it deforms an affected cloud beyond tolerance or recoverability threshold.

## PO-005 — Threshold Record

Every actualized crossing requires a historical shell and a propagated record.

## PO-006 — Observer Projection

The same radial tessellation may appear smooth or cell-resolved depending on observer distance, resolution, noise, lag, and probe capacity.

## Done Criteria

The initial CHF package is done when `chf_validation_run.yml` uploads a report showing all three CHF specs pass their expected ALLOW / DENY / FAIL_CLOSED outcomes.
