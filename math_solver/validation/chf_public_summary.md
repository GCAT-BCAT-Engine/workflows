# Public Summary — Consequence Horizon Formalism

StegVerse Consequence Horizon Formalism validates whether a consequence-bearing transition has standing to bind reality at commit.

The point is not merely whether an AI agent, tool, script, workflow, or human operator can perform an action.

The point is whether the action remains admissible when it becomes consequential.

CHF models that boundary through deterministic gates covering:

```text
state admissibility
lag and uncertainty
record and receipt sufficiency
authority drift
policy version validity
external effects
rollback and repair
participant recoverability
privacy and consent
temporal coherence
deterministic replay
ecosystem rejoin
multi-node consensus
quarantine
supersession
cross-repository ingestion
production readiness
```

The current validation lane has passed `chf-001` through `chf-040` in GitHub Actions.

This establishes a public proof surface for the StegVerse claim that governance should be enforced at the execution boundary, not merely reviewed after the fact.

## One-line version

```text
StegVerse CHF tests whether a transition has standing to bind consequences before it enters reality.
```

## Longer version

```text
A system is not governed because it can explain what happened.
It is governed when it can prove, before commit, that the transition still has authority,
recordability, recoverability, policy validity, participant safety, temporal coherence,
and replayable evidence sufficient to bind consequences.
```
